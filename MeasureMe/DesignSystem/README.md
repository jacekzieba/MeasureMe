# MeasureMe Design System

## Control Styles

| Style | Role | When to use |
| --- | --- | --- |
| `AppCTA` | Primary action | Use for the main screen-level action that moves the user forward or commits the expected task. |
| `AppSecondary` | Secondary action | Use for non-destructive alternatives, dismissals, and supporting actions that should not compete with the primary CTA. |
| `AppDestructive` | Destructive confirmation | Use for irreversible or data-removing actions after intent is clear. |
| `LiquidCapsule` | Compact contextual action | Use for small inline actions, filters, and lightweight controls near content. |
| `PhotoTagChip` | Selectable tag or filter | Use for photo tags and chip-style toggles where selected state matters. |
| `GlassSegmented` | Segmented choice | Use for mutually exclusive options that should read as one grouped control. |

## Glass Depth

Use glass depth to communicate hierarchy:

| Depth | Role | Examples |
| --- | --- | --- |
| `floating` | Hero | Home hero, feature hero, or the dominant card on a screen. |
| `elevated` | Primary content | Main forms, primary modules, and focused content groups. |
| `base` | Inline or list | Rows, summaries, nested supporting cards, and compact list content. |
